  ---
  🔬 Dan Koe GEO 方法论拆解

  核心逻辑链

  传统 SEO:  写文章 → 堆关键词 → 买外链 → Google 排名 ↑
  GEO 方法:  深度内容 → AI 可摘取模块 → AI 引用你 → Google 认为你是权威 → 排名 ↑

  关键洞察：Google 排名是果，AI 引用是因。 当 Gemini/Perplexity 等 AI
  引擎频繁引用你的内容时，Google 的算法会将你视为"权威来源"。

  四步法的本质拆解

  ┌──────────┬─────────────────┬─────────────────────────┬─────────────────┐
  │   步骤   │     做什么      │       为什么有效        │  对应 GEO 原理  │
  ├──────────┼─────────────────┼─────────────────────────┼─────────────────┤
  │ 1. 竞品  │ 6 个竞品页面 →  │ 找到"所有人都在说但没人 │ AI 偏好独特增量 │
  │ 双杀拆解 │ Claude          │ 说透"的内容缺口         │ 信息            │
  │          │ 找共同缺失      │                         │                 │
  ├──────────┼─────────────────┼─────────────────────────┼─────────────────┤
  │ 2.       │ 8 组 FAQ Schema │ 让 AI 能直接"摘"你的答  │ AI 引用时优先选 │
  │ 生成可摘 │  + 4 类模块     │ 案，无需改写            │ 取结构化短答案  │
  │ 取模块   │                 │                         │                 │
  ├──────────┼─────────────────┼─────────────────────────┼─────────────────┤
  │ 3. Frase │ 补缺失关键词但  │ SEO 和 GEO              │ 同时满足爬虫和  │
  │          │ 不破坏短段落    │ 双修，不牺牲可读性      │ LLM             │
  │ 语义优化 │                 │                         │                 │
  ├──────────┼─────────────────┼─────────────────────────┼─────────────────┤
  │ 4.       │ 分开监控 SEO    │ 防止单向优化导致另一种  │                 │
  │ Topify   │ 流量 + AI 引用  │ 流量下降                │ 数据驱动迭代    │
  │ 追踪     │                 │                         │                 │
  └──────────┴─────────────────┴─────────────────────────┴─────────────────┘

  四个 GEO 内容模块的定义

  1. 开篇答案块 (Opening Answer Block) — 页面最顶部，一句话直接回答问题。AI
  抓取时最优先摘取。
  2. 排除声明 (Exclusion Disclaimer) —
  页面中间，"谁不适合/什么情况下不要用"。增加内容可信度和覆盖面。
  3. 条件对比表 (Conditional Comparison) — "如果你 X，选 A；如果你 Y，选
  B"。这是竞品分析中最常见的缺失模块。
  4. FAQ Schema (结构化问答) — 页面末尾，8 组 Q&A，标记 FAQ Schema。AI
  直接摘取作为 featured snippet。
